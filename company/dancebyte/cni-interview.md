# Kubernetes CNI 面试题（含参考答案）

## 基础题

### 1）CNI 是什么？和 CSI、CRI 有什么区别？
**答：**
- CNI 负责容器网络连接（IP、路由、连通性、部分策略）。
- CRI 负责容器运行时管理（创建/启动/停止容器）。
- CSI 负责存储卷生命周期（供给、挂载、扩容、快照）。

### 2）Pod 网络模型核心要求是什么？
**答：**
- 每个 Pod 有独立 IP。
- Pod 间无需 NAT 可直接通信（集群内）。
- 同 Pod 内容器共享网络命名空间。

### 3）CNI 在 Pod 启动阶段做了什么？
**答：**
1. 创建 veth pair。
2. 一端放入 Pod netns，另一端接入宿主机网络。
3. 通过 IPAM 分配 Pod IP。
4. 配置路由、转发和必要规则。

### 4）什么是 IPAM？
**答：**
- IP Address Management，负责 Pod IP 的分配、回收与冲突管理。
- 常见问题包括地址池耗尽、IP 泄漏、地址重叠。

### 5）NetworkPolicy 生效依赖什么？
**答：**
- 依赖 CNI 插件支持策略能力（如 Calico）。
- 规则基于标签和命名空间选择器匹配，不是基于 Pod 名称硬编码。

## 进阶题

### 6）Overlay 与 BGP 路由模式怎么选？
**答：**
- Overlay：部署简单，适配性好，但有封装开销。
- BGP 路由：路径短、性能好，但网络规划和运维复杂度较高。
- 选择取决于底层网络可控性、性能目标、团队运维能力。

### 7）为什么会出现跨节点网络不通？
**答：**
- 隧道/VXLAN 异常、路由发布异常、节点转发参数不正确。
- MTU 不一致导致分片或丢包。
- 网络策略或安全组误拦截。

### 8）如何定位是 CNI 问题还是应用问题？
**答题思路：**
1. 先验证 Pod IP 和 Service/DNS。
2. 再用同节点/跨节点连通性测试排除应用层。
3. 查看 CNI 日志和节点路由/iptables。
4. 最后回应用日志确认是否为业务协议问题。

### 9）什么是 Hairpin、SNAT、DNAT，面试中怎么讲？
**答：**
- SNAT：源地址转换，常用于出网。
- DNAT：目标地址转换，常用于 Service 转发。
- Hairpin：Pod 访问自身 Service 再回到本 Pod 的回环场景，依赖正确网络配置。

### 10）网络策略默认是“拒绝全部”吗？
**答：**
- 不是全局默认拒绝。
- 某命名空间/Pod 一旦被策略选中，未放行方向会被拒绝；未被策略选中的通常仍是默认放通（依实现细节）。

## 场景题

### 11）集群偶发 DNS 超时，你怎么排查？
**答题思路：**
- 看 CoreDNS Pod 状态、资源和日志。
- 看 Pod 到 CoreDNS Service 的链路和策略放行。
- 检查节点 conntrack、UDP 丢包、MTU 与底层网络质量。

### 12）业务反馈“只有跨节点访问慢”，你会怎么做？
**答题思路：**
- 对比同节点与跨节点 RTT/吞吐。
- 检查是否走隧道封装、MTU 是否合理。
- 观察节点网络带宽、丢包和 CPU softirq 压力。
- 评估是否调整为路由直连/BGP 或优化拓扑调度。

## Calico / Flannel / Canal / Multus 专项

### 13）Calico 的核心特点是什么？
**答：**
- 提供强网络策略能力（NetworkPolicy/GlobalPolicy 等）。
- 支持多种数据平面与路由模式（如 BGP、VXLAN）。
- 适合对安全隔离和策略治理要求高的生产环境。

### 14）Flannel 的定位是什么？
**答：**
- 重点解决“Pod 跨节点连通”，实现简单、部署轻量。
- 更偏基础网络覆盖，策略能力通常需搭配其他方案。
- 适合快速搭建和对策略要求不高的场景。

### 15）Canal 是什么？为什么会用它？
**答：**
- Canal = Flannel + Calico 组合方案。
- 用 Flannel 提供基础网络连通，用 Calico 提供策略能力。
- 常用于想保留 Flannel 网络简单性，同时补齐策略控制的场景。

### 16）Multus 解决了什么问题？
**答：**
- 让 Pod 能挂多个网络接口（多 CNI）。
- 典型场景：默认业务网 + 存储网/管理网/DPDK/SR-IOV 专用网。
- 适用于 NFV、AI 高性能网络、存储分流等需求。

### 17）Calico 和 Flannel 如何选型？
**答：**
- 要强策略治理、多租户隔离、可观测性：优先 Calico。
- 要快速、轻量、只需基础连通：Flannel 成本更低。
- 如果历史包袱是 Flannel，但要策略：可考虑 Canal。

### 18）Multus 引入后运维复杂度体现在哪？
**答：**
- IP 地址规划复杂（多网段、多 IPAM）。
- 故障域增多（默认网正常但附加网异常）。
- 排障要分接口、分网络逐层验证，监控体系也要扩展。

### 19）Calico 常见故障怎么排查？
**答题思路：**
1. 看 Felix/BGP 组件状态与日志。
2. 看策略是否误匹配标签。
3. 看节点路由发布和可达性。
4. 看底层网络与 MTU。

### 20）Flannel 常见故障怎么排查？
**答题思路：**
1. 看 flanneld 状态与子网分配。
2. 检查 VXLAN 设备/隧道状态。
3. 检查节点转发与 iptables 规则。
4. 排查底层网络丢包与 MTU。

### 21）Canal 方案最容易踩的坑是什么？
**答：**
- 网络组件职责边界不清，导致问题归因困难。
- 升级时组件版本兼容性需要严格验证。
- 策略生效问题常与标签、命名空间选择器误配有关。

### 22）Multus 场景下面试官常追问什么？
**答：**
- 第二网卡的 IPAM 怎么做（host-local、whereabouts 等）。
- 应用如何选择出接口（路由策略、绑定网卡）。
- 多网卡下服务发现与安全策略如何统一治理。

## 高频追问（可直接背）

### 追问 1：为什么说“CNI 选型本质是网络模型选型”？
- 因为最终决定的是数据路径、性能上限、策略能力和运维复杂度。

### 追问 2：怎样体现你有生产排障经验？
- 给出分层路径：对象状态 -> CNI 组件日志 -> 节点网络栈 -> 底层网络。
- 给出量化指标：丢包率、RTT、重传率、conntrack 使用率。

### 追问 3：如何降低网络变更风险？
- 灰度变更、分批节点升级、保留回滚方案。
- 变更前后压测与关键链路探测对比。

## 速记版（1 分钟）

- CNI 负责 Pod 网络连接；CRI 管运行时；CSI 管存储。
- 主流程：`ADD` 时配 veth/IP/route，`DEL` 时回收资源。
- Calico 强策略，Flannel 轻连通，Canal 是组合，Multus 做多网卡。
- 常见故障：跨节点不通、DNS 超时、策略误杀、MTU 不匹配。
- 排障顺序：对象事件 -> CNI 日志 -> 节点网络栈 -> 底层网络。




## CNI 网络 面试题 + 答案
1. 常见 CNI 有哪些？
答：Calico、Flannel、Cilium、Canal、Weave、Kube-OVN、Antrea、Multus
2. CNI 作用是什么？
答：K8s 容器网络标准接口，统一给 Pod 分配 IP、打通节点内外网络、实现 Pod 间通信。
3. Calico 原理简单说
答：基于 BGP 路由，每个节点维护路由表，支持网络策略、隔离、ACL，性能高，生产最常用。
