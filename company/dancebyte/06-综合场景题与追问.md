# 综合场景题与追问（含参考答案）

## 场景 1：接口 QPS 激增，响应时间翻倍，怎么排查？
**参考回答：**
1. 先看监控：QPS、P95/P99、错误率、线程池、连接池  
2. 再看瓶颈：DB 慢查询、缓存命中率、下游超时  
3. 快速止血：限流、降级、缓存兜底  
4. 长期治理：热点拆分、异步化、索引优化

**追问：为什么先看监控不是先看日志？**  
先定故障时间窗和范围，再用日志做深挖，效率更高。

## 场景 2：K8s 上服务频繁重启，业务间歇不可用
**参考回答：**
1. 看 Pod 事件和退出码（OOMKilled / CrashLoopBackOff）  
2. 看探针配置是否过严  
3. 看资源请求/限制是否合理  
4. 看依赖是否可达（DB/Redis/DNS）  
5. 修正后做滚动发布验证

## 场景 3：缓存命中率骤降，数据库压力暴涨
**参考回答：**
1. 查缓存 key 设计是否变化  
2. 查过期策略是否集中失效  
3. 检查是否出现穿透/击穿  
4. 先加保护：空值缓存、互斥重建、限流  
5. 再修复：分层缓存与过期时间随机化

## 场景 4：调度引擎经常把任务分到“慢节点”
**参考回答：**
1. 补节点质量标签（性能、负载、网络）  
2. 优化打分权重（历史失败率、资源余量）  
3. 引入回放与仿真评估策略  
4. 增加调度后反馈闭环（成功率、耗时）

## 场景 5：让你重构一个“耦合严重”的后端服务
**参考回答：**
1. 先识别边界：领域拆分与依赖拓扑  
2. 定义契约：API/事件模型  
3. 分阶段迁移：旁路写、灰度切流  
4. 加可观测性：关键路径指标与告警  
5. 设回滚策略：每阶段可回退

## 场景 6：面试官让你设计“云平台资源创建 API”
**参考回答结构：**
1. 请求模型：幂等键 + 参数校验  
2. 执行模型：异步任务 + 状态机  
3. 一致性：最终一致 + 补偿机制  
4. 可观测性：trace + 任务日志 + 状态事件  
5. 安全：权限、配额、审计

## 高频追问模板

### Q：你这套方案最大的风险是什么？
**答：**复杂度和运维成本。通过分阶段上线、灰度与回滚、可观测性来控制风险。

### Q：如果只有 2 周你会先做什么？
**答：**先交付最小可用闭环：核心路径稳定 + 关键监控告警 + 明确回滚。

### Q：如何证明你的优化有效？
**答：**用对比指标：P95、错误率、吞吐、资源利用率，发布前后 A/B 数据说话。

## 面试官二连追问（加强版）

### 追问 1：如果你的首选方案上线失败，Plan B 是什么？
**答题要点：**
- 给回滚路径（版本、流量、配置）
- 给降级策略（限流、只读、兜底缓存）
- 给恢复窗口（分钟级目标）

### 追问 2：你如何向非技术同学解释这次故障？
**答题要点：**
- 先讲影响范围和时长
- 再讲根因和立即修复
- 最后讲防复发动作和时间表

### 追问 3：你怎么做复盘避免“下次还犯”？
**答题要点：**
- 责任无责化，聚焦系统问题
- 行动项必须有 owner 和 deadline
- 把监控、告警、演练纳入日常机制
