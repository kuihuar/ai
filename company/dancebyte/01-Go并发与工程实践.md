# Go 并发与工程实践面试题（含参考答案）

## 1. goroutine 和线程的核心区别？
**答：**goroutine 是用户态轻量协程，由 Go runtime 调度；线程由 OS 调度。goroutine 启动成本低、切换快，适合高并发 I/O 场景。

## 2. GMP 调度模型是什么？
**答：**G=goroutine，M=内核线程，P=处理器上下文。P 绑定 M 执行 G，支持 work stealing，提升多核利用率。

## 3. channel 的本质和使用原则？
**答：**channel 是线程安全队列 + 同步原语。用于 goroutine 间通信。原则是“不要通过共享内存通信，应该通过通信共享内存”。

## 4. mutex 和 channel 如何选？
**答：**共享状态读写保护优先 mutex/RWMutex；异步任务编排、流水线、消息传递优先 channel。

## 5. context 常见用途？
**答：**超时控制、取消传播、链路元信息透传。服务端必须把请求 context 传到 DB、RPC、缓存调用层。

## 6. 如何避免 goroutine 泄漏？
**答：**确保 goroutine 有退出条件；配合 context cancel；避免无限阻塞读 channel；后台任务要有 stop 机制。

## 7. sync.Pool 适合解决什么问题？
**答：**降低高频对象分配和 GC 压力，适合短生命周期、可复用对象（如 bytes.Buffer）。

## 8. Go 常见性能优化手段？
**答：**pprof 定位热点；减少逃逸；预分配切片容量；减少反射；对象复用；降低锁竞争；避免过度 goroutine。

## 9. Gin / Echo 的中间件链路要注意什么？
**答：**统一鉴权、日志、trace、panic recover；错误码标准化；中间件不要做重 CPU 操作。

## 10. gRPC + Protobuf 优势？
**答：**IDL 强约束、序列化效率高、接口演进可控（向后兼容字段）、跨语言支持好，适合内部服务通信。

## 高频追问（含回答要点）

### 追问 1：为什么你说 channel 比锁更“优雅”，但生产里还是大量用锁？
**要点：**
- channel 适合“任务编排/消息流转”
- 锁适合“共享状态保护”
- 生产以性能和可维护性为先，不迷信单一范式

### 追问 2：如何证明并发优化真的有效？
**要点：**
- 给基准：QPS、P95、CPU、GC 次数
- 用 pprof 对比优化前后热点
- 压测环境和数据规模要可复现

### 追问 3：context 传递最容易踩什么坑？
**要点：**
- 忘记传到下游导致取消失效
- 把大对象塞进 context
- 没有在 goroutine 中监听 Done 退出
