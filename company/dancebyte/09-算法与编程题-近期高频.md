# 算法与编程题（近期高频，Go 后端面试向）

> 说明：这份不是“刷题大全”，而是后端面试最容易出现的题型集合。
> 重点：数组/哈希、滑动窗口、堆、二分、链表、并发小题。

## 1. 高频算法题型清单

## 1.1 数组 + 哈希（超高频）

1. 两数之和（Two Sum）
2. 子数组和等于 K（LeetCode 560）
3. 最长连续序列
4. 字母异位词分组

**考点**：哈希映射、前缀和、O(n) 优化思路。

## 1.2 滑动窗口（高频）

1. 无重复字符的最长子串
2. 找到字符串中所有字母异位词（LeetCode 438）
3. 最小覆盖子串

**考点**：窗口扩张/收缩、计数器、边界处理。

## 1.3 TopK / 堆（高频）

1. 前 K 个高频元素（LeetCode 347）
2. 合并 K 个有序链表
3. 数据流中位数

**考点**：最小堆/最大堆、复杂度 O(n log k)。

## 1.4 链表与双指针（中高频）

1. 反转链表
2. 环形链表判定
3. 删除倒数第 N 个节点

**考点**：快慢指针、原地操作、边界节点。

## 1.5 二分与搜索（中高频）

1. 二分查找模板
2. 搜索旋转排序数组
3. 第一个/最后一个目标值位置

**考点**：左闭右闭/左闭右开区间一致性。

## 6. 每题升级版（标准答案 + 追问 + 反问陷阱）

### Q1：两数之和（Two Sum）

- 标准答案：一次遍历，`map[value]index` 记录已访问元素。对当前 `x` 查 `target-x` 是否存在，存在即返回。时间 O(n)，空间 O(n)。
- 高频追问：为什么不用排序 + 双指针？
- 追问要点：排序后可 O(n log n) 求值但会丢原始下标（除非额外结构），题目通常要求返回原下标。
- 反问陷阱：先写双重循环再说优化，容易被判断缺乏复杂度敏感度。

### Q2：子数组和等于 K（前缀和 + 哈希）

- 标准答案：维护前缀和 `pre`，若存在历史前缀和 `pre-k`，则说明有子数组和为 k。用 map 统计前缀和出现次数。时间 O(n)。
- 高频追问：为什么 map 要先放 `0:1`？
- 追问要点：处理从下标 0 开始就满足条件的子数组。
- 反问陷阱：把“最长子数组”与“子数组个数”写成同一套更新逻辑，统计会错。

### Q3：无重复字符最长子串（滑动窗口）

- 标准答案：右指针扩张窗口，字符频次超限时左指针收缩直到合法；每轮更新最大长度。时间 O(n)。
- 高频追问：字符集很大怎么办？
- 追问要点：优先用 map 记录最近位置；ASCII 可用数组提速。
- 反问陷阱：左指针每次只移动一步却不循环收缩，会漏掉非法窗口。

### Q4：前 K 个高频元素（堆）

- 标准答案：先统计频次，再维护大小为 k 的最小堆，堆顶是当前第 k 高频。整体 O(n log k)。
- 高频追问：什么时候用快速选择优于堆？
- 追问要点：离线一次性计算、且可接受平均复杂度时可用 quickselect 近似 O(n)。
- 反问陷阱：k 接近 n 仍坚持堆，实际收益不明显。

### Q5：合并 K 个有序链表

- 标准答案：最小堆存每个链表当前节点，弹出最小后推进其 next。总复杂度 O(N log k)（N 为总节点数）。
- 高频追问：分治合并方案呢？
- 追问要点：两两合并也是 O(N log k)，实现有时更直观。
- 反问陷阱：忽略空链表和单链表输入，代码健壮性扣分。

### Q6：二分查找模板怎么避免死循环？

- 标准答案：统一区间定义（推荐左闭右闭），循环条件用 `l <= r`，中点用 `mid := l + (r-l)/2`，并严格保证每次更新都缩小区间。
- 高频追问：找第一个 >= target 怎么写？
- 追问要点：命中后继续向左收缩（`r = mid - 1`）并记录答案。
- 反问陷阱：区间定义混用（左闭右开 + `<=`），边界极易错。

### Q7：LRU 缓存怎么设计？

- 标准答案：`hash map + doubly linked list`。map O(1) 定位节点，链表 O(1) 维护最近使用顺序；`get/put` 都是 O(1)。
- 高频追问：并发场景怎么办？
- 追问要点：全局锁简单但竞争高；可分片（shard）降低锁粒度，或直接使用成熟缓存库。
- 反问陷阱：只会背结构，不会说明淘汰时机与容量策略。

### Q8：实现协程池的关键点

- 标准答案：固定 worker 数 + 有界任务队列 + 关闭协议（`context`/`close`）+ panic 保护。核心目标是限并发和可控资源回收。
- 高频追问：队列满了怎么办？
- 追问要点：阻塞等待、超时丢弃、降级返回三种策略，按业务 SLA 选。
- 反问陷阱：只写“无限队列”，高峰时会放大内存风险。

### Q9：令牌桶限流如何落地？

- 标准答案：按固定速率补充令牌，请求先拿令牌再执行。支持突发流量且平均速率可控；可本地限流或 Redis 分布式限流。
- 高频追问：分布式限流时钟不一致怎么办？
- 追问要点：集中式时间源或基于 Redis Lua 原子计数避免多节点漂移影响。
- 反问陷阱：把漏桶和令牌桶说成一样。两者平滑与突发能力不同。

### Q10：100G 文件统计词频怎么做？

- 标准答案：先按 hash 分片到多个小文件（保证同词同分片），每个分片并行统计，再归并 topN。必要时使用外排序处理内存限制。
- 高频追问：如何控制热点词导致的分片倾斜？
- 追问要点：增加分片数、二级分桶、热点词单独通道。
- 反问陷阱：单机一次性读入内存统计，不符合数据规模约束。
