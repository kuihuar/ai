# IaaS 与资源调度面试题（含参考答案）

## 1. IaaS 控制面的核心职责？
**答：**资源抽象、生命周期编排、状态管理、权限与配额、策略下发、审计。

## 2. 数据面在云平台中做什么？
**答：**承载实际流量与执行动作，如网络转发、存储读写、代理通道等。

## 3. 资源调度引擎关键能力？
**答：**资源打分、亲和/反亲和、容量约束、优先级、抢占、公平性。

## 4. 如何设计调度策略可扩展？
**答：**插件化评分器 + 策略配置化 + 回放测试 + 可观测性埋点。

## 5. 控制面高可用怎么做？
**答：**无状态化、主从选举、幂等 API、重试退避、状态持久化与回放。

## 6. 资源创建慢如何优化？
**答：**异步任务化、并行依赖创建、缓存热点信息、减少串行 RPC、预热池化。

## 7. 配额系统怎么设计？
**答：**租户维度配额（CPU/内存/存储/IP），支持硬限额与软配额，配合审批流程。

## 8. 如何保证 API 幂等？
**答：**请求幂等键 + 资源唯一约束 + 幂等状态机 + 重试可重入。

## 9. 调度失败如何反馈给用户？
**答：**标准化失败原因（资源不足/策略冲突/依赖未就绪），并给出可操作建议。

## 10. 资源生命周期要覆盖哪些状态？
**答：**Pending、Creating、Running、Updating、Deleting、Error、Deleted（并明确转移条件）。

## 高频追问（含回答要点）

### 追问 1：调度策略如何避免“局部最优”？
**要点：**
- 不能只看当前空闲率
- 要加入历史失败率与节点健康权重
- 定期做策略回放校准

### 追问 2：资源创建 API 为什么建议异步化？
**要点：**
- 下游依赖多且耗时不可控
- 同步链路容易超时级联
- 异步 + 状态机更适合云资源生命周期

### 追问 3：控制面重启后如何保证任务不丢？
**要点：**
- 状态持久化（DB/etcd）
- 幂等重放与任务恢复
- 明确任务租约和超时补偿
