## Data Structure
数据结构是计算机中组织和存储数据的方式

- Array
- Stack / Queue
- Priority Queue(heap)
- Linked List(singly/doubly)
- Tree / Binary Tree / Binary Search Tree
- Hash Table
- Disjoint Set
- Trie
- Bloom Filter
- LRU Cache




### 一、基础数据结构(线性)
1. 数组 (Array)

- 连续内存存储，支持快速随机访问（O(1)）。

- 缺点：大小固定，插入/删除效率低（O(n)）。

2. 链表 (Linked List)

- 节点通过指针连接，分为单链表、双向链表、循环链表。

- 优点：动态大小，插入/删除快（O(1)）。

- 缺点：随机访问慢（O(n)），内存开销大（存储指针）。

3. 栈 (Stack)

- 后进先出（LIFO），操作：push（入栈）、pop（出栈）。

- 应用：函数调用栈、括号匹配、表达式求值。

4. 队列 (Queue)

- 先进先出（FIFO），操作：enqueue（入队）、dequeue（出队）。

- 变种：双端队列（Deque）、优先队列（Priority Queue）。

### 二、非线性数据结构
5. 树 (Tree)

- 二叉树：每个节点最多两个子节点。

  - 二叉搜索树（BST）：左子树值 < 根 < 右子树值。

  - 平衡树：如 AVL 树、红黑树，优化查找效率。

- 堆 (Heap)：完全二叉树，分为最大堆（根最大）和最小堆（根最小），用于优先队列。

- Trie（字典树）：高效存储字符串集合，前缀匹配场景（如自动补全）。

6. 图 (Graph)

- 由节点（顶点）和边组成，分为有向图和无向图。

- 存储方式：邻接矩阵、邻接表。

- 应用：社交网络、路径规划（Dijkstra 算法）。

7. 哈希表 (Hash Table)

- 通过哈希函数将键映射到值，支持平均 O(1) 的插入、删除、查找。

- 缺点：哈希冲突（解决方法：开放寻址、链地址法）。
### 三、高级数据结构
8. 跳表 (Skip List)

- 多层链表，通过概率平衡实现近似 O(log n) 的查询效率，用于 Redis 的有序集合。

9. 并查集 (Disjoint Set Union, DSU)

- 管理元素分组，支持合并（Union）和查询（Find）操作，用于连通性问题。

10. B树/B+树

- 多路平衡搜索树，用于数据库和文件系统（减少磁盘 I/O）。

11. 布隆过滤器 (Bloom Filter)

- 概率型数据结构，判断元素是否可能存在或一定不存在，用于缓存穿透防护。

12. 线段树 (Segment Tree)

- 支持区间查询和更新，时间复杂度 O(log n)，用于范围统计问题（如区间求和）。

### 四、其他数据结构
13. 位掩码 (Bitmask)

- 用二进制位表示状态，节省内存（如权限管理）。

14. 矩阵 (Matrix)

- 二维数组，用于图像处理、科学计算。

15. 缓存结构

- LRU（最近最少使用）、LFU（最不经常使用）等缓存淘汰策略。

#### 如何选择数据结构？
- 访问模式：频繁随机访问 → 数组；频繁插入删除 → 链表。

- 数据关系：层级关系 → 树；网络关系 → 图。

- 性能需求：快速查找 → 哈希表；范围查询 → B+树。

- 内存限制：紧凑存储 → 位掩码；动态扩展 → 链表。




#### 线性/非线性结构

- 线性结构应该是指元素按顺序排列，每个元素前后只有一个直接前驱或后继，比如数组、链表、栈、队列这些。
- 非线性结构的话，元素之间可能有多个连接，像树和图，每个节点可能有多个子节点或多个边

##### 一、线性数据结构

###### 特点：
- 单一顺序关系

  - 元素之间只存在一对一的线性关系，每个元素仅有一个直接前驱和一个直接后继（除了首尾元素）。

  - 例如：数组中的元素按索引依次排列；链表中节点通过指针单向或双向连接。

- 存储方式

  - 可以是连续内存（如数组）或离散内存（如链表）。

- 访问模式

  - 支持顺序访问（如遍历数组），某些结构支持快速随机访问（如数组通过索引）。

###### 常见结构：
- 数组、链表、栈、队列、哈希表（虽然哈希函数是非线性映射，但底层通常用数组实现）。

###### 典型场景：
- 需要严格顺序操作的场景，如文本编辑器的撤销操作（栈）、CPU 任务调度（队列）。


##### 二、非线性数据结构
###### 特点：
- 复杂关系

  - 元素之间存在一对多或多对多的关系。

  - 例如：树中一个父节点可能有多个子节点；图中节点之间可以任意连接。

- 存储方式

  - 通常需要更复杂的结构，如指针（树）、邻接表/矩阵（图）。

- 访问模式

  - 一般不支持直接随机访问，需通过特定算法遍历（如树的深度优先搜索、图的广度优先搜索）。

###### 常见结构：
- 树（二叉树、堆、B树）、图、并查集、字典树。

###### 典型场景：
- 层级关系：文件系统目录（树）、数据库索引（B+树）。

- 网络关系：社交网络好友链（图）、路由器拓扑（图）。

##### 三、对比表格
|特性|线性数据结构|非线性数据结构|
|---|---|---|
|元素关系|一对一（单线顺序）|一对多或多对多|
|内存占用|通常更紧凑（如数组）|通常更高（需存储额外指针/边信息）|
|访问效率|随机访问快（数组）|依赖遍历算法（如DFS/BFS）|
|典型操作|增删查改（按顺序或索引）|递归遍历、路径查找、平衡操作|
|应用场景|简单列表、缓存、序列化数据|层级管理、网络关系、复杂查询|


##### 四、为什么需要非线性结构？
1. 解决复杂关系

  - 线性结构无法高效表示层级（如公司组织架构）或网络关系（如地图路径）。

2. 优化特定操作

  - 例如：树结构（如二叉搜索树）将查找时间复杂度从线性结构的 O(n) 降低到 O(log n)。

3. 节省存储空间

  - 非线性结构（如字典树）可通过共享前缀压缩存储大量字符串。
